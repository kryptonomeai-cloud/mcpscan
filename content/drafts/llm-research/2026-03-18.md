## New Releases

### Mistral Small 4 (119B, 6.5B active) — Released ~Mar 16-17
- **Architecture**: MoE with 128 experts, 4 active per token
- **Context**: 256K tokens
- **Multimodal**: Text + image input, text output
- **Key innovation**: Unified model replacing three separate families (Instruct, Magistral reasoning, Devstral coding). Per-request `reasoning_effort` toggle (none/high)
- **Performance**: Matches/surpasses "GPT-OSS 120B" on AIME, LiveCodeBench while generating significantly shorter outputs (20% less on LiveCodeBench, 3.5-4× less on AA LCR vs Qwen)
- **Efficiency**: 40% reduction in end-to-end completion time (latency-optimized), 3× more requests/sec (throughput-optimized) vs Mistral Small 3
- **Extras**: Eagle head for speculative decoding, NVFP4 quantization checkpoint available
- **License**: Apache 2.0
- **HuggingFace**: `mistralai/Mistral-Small-4-119B-2603` — trending, uploaded ~12 hours ago
- **Relevance**: This is a strong local model candidate. At 6.5B active params the MoE architecture is efficient, but 119B total means ~60GB+ in FP8. Would need 3-4× RTX 3090 (24GB each) for full precision, or NVFP4 quantization to fit on fewer. Worth testing.

### Leanstral (119B, 6.5B active) — Released ~Mar 17
- Built on Mistral Small 4 architecture
- First open-source code agent for **Lean 4** theorem proving
- Integrated with Mistral Vibe (their coding agent)
- Apache 2.0 license
- **HuggingFace**: `mistralai/Leanstral-2603`
- Niche but notable for formal verification / math proof workloads

### Mistral Forge — Announced Mar 17
- Enterprise system for building "frontier-grade AI models grounded in proprietary knowledge"
- Details sparse — appears to be a managed fine-tuning/RAG platform for enterprises
- Signals Mistral pivoting harder to enterprise monetization

## Cloud Provider News

### NVIDIA NemoClaw — Announced GTC Mar 16
- **What**: Security and privacy wrapper for OpenClaw using NVIDIA Agent Toolkit
- **How**: Installs Nemotron models + OpenShell runtime in a single command, adds policy-based security, network, and privacy guardrails
- **Runs on**: Any dedicated platform — RTX PCs, RTX PRO workstations, DGX Station, DGX Spark, cloud
- **Jensen quote**: "Mac and Windows are the operating systems for the personal computer. OpenClaw is the operating system for personal AI."
- **Peter Steinberger quote**: "With NVIDIA and the broader ecosystem, we're building the claws and guardrails that let anyone create powerful, secure AI assistants."
- **Build-a-claw event** at GTC Park March 16-19
- **Relevance**: Directly relevant to our OpenClaw setup. NemoClaw could add sandbox security to our Mac mini deployment with local Nemotron models on the GPU server.

## Relevance to Us (Mac mini + 5× RTX 3090 GPU server)

### Worth Testing
1. **Mistral Small 4 NVFP4** (`mistralai/Mistral-Small-4-119B-2603-NVFP4`): At FP4 quantization, ~30GB. Could potentially fit on 2× RTX 3090. The unified instruct/reasoning/code model with per-request reasoning effort is very appealing for an OpenClaw backend. Apache 2.0.
2. **NemoClaw**: Directly relevant — adds sandbox security to OpenClaw. Worth investigating if it works with our GPU server setup. Could run Nemotron 3 Super locally with the privacy guardrails.

### Watch List
- **Mistral Forge**: If it enables fine-tuning Mistral Small 4 on our own data cost-effectively, could be valuable.
- **Meta Avocado**: Delayed to May but if it's open-weight like Llama, could be significant.

### No Action Needed
- Leanstral: Too niche (Lean 4) unless we get into formal verification
- HumeAI TADA: Nice TTS but we have sag/ElevenLabs already
- IBM Granite Speech: Good ASR but not our current need
