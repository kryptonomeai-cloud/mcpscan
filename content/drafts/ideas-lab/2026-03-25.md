### 🥈 Second Idea: UK Cyber Essentials MFA Compliance Service

The UK's Cyber Essentials scheme is getting a **major update on April 27, 2026** that creates immediate consulting demand:

**What's changing:**
- MFA is now **mandatory on ALL cloud services** — failure to implement = automatic certification failure
- New definition: ANY service accessed via a business email/account is in-scope (including free-tier SaaS)
- If a service offers MFA only as a paid add-on, you **must pay for and enable it**
- Shadow IT apps without MFA = non-compliance

**The problem:** Most UK businesses have 15x more cloud accounts per employee than they think. They have no visibility into which apps their staff actually use, let alone which have MFA enabled.

**The service:**
1. **SaaS Discovery Audit** — scan and catalogue all cloud services in use (shadow IT included)
2. **MFA Gap Analysis** — identify which services lack MFA, which need upgrades to paid tiers
3. **Implementation Sprint** — configure MFA across all services, set up SSO where possible
4. **Certification Preparation** — help pass the updated Cyber Essentials assessment

**Revenue Potential:** £1,000–3,000 per SME engagement. Volume play — there are ~90,000 organisations with active Cyber Essentials certifications, and most will need help adapting. April 27 deadline creates urgency.

**Quick Win:** Write a blog post/LinkedIn article: "Your Cyber Essentials Certification Will Fail in April — Here's What Changed" targeting UK business owners.

## 2. Notable Finds

### InstallFix: A New Social Engineering Category
Push Security identified "InstallFix" attacks — attackers clone legitimate CLI tool installation pages (specifically **Claude Code** and **NotebookLM**) and serve them via malvertised search results. Victims copy what looks like a normal `curl | bash` install command but execute malware instead. This is particularly dangerous because:
- AI tools are attracting less security-conscious "vibe coders"
- The pages are pixel-perfect clones
- The attack requires no social engineering pretext — installing software IS the pretext

**Opportunity:** Create a browser extension or CLI tool that validates `curl | bash` commands against known-good checksums before execution. Or write educational content targeting the growing vibe-coder audience about terminal security basics.

### AWS Bedrock: 8 Attack Vectors Mapped
XM Cyber published detailed research on 8 attack vectors inside AWS Bedrock, covering:
1. Model invocation log attacks (S3 redirect for silent data harvest)
2. Knowledge base data source attacks (steal SaaS credentials, lateral move to AD)
3. Knowledge base data store attacks (vector DB credential theft)
4. Direct agent attacks (rewrite prompts, attach malicious executors)
5. Indirect agent attacks (Lambda code injection)
6. Flow attacks (inject sidecar nodes, bypass business rules)
7. Guardrail attacks (weaken or delete safety filters)
8. Managed prompt attacks (poison templates across all agents)

**Key insight:** All 8 vectors target permissions and configurations AROUND the model, not the model itself. A single over-privileged identity is enough. This is the attack surface that Guardian Agent technology is trying to address.

## 3. Quick Win: Publish a Cyber Essentials MFA Compliance Checklist (2 hours)

The April 27 deadline is ~33 days away. Create a free, downloadable PDF checklist:

1. **Audit your SaaS stack** — list every cloud service accessed with business credentials
2. **Check MFA status** — for each service, is MFA enabled? Available? Paid-only?
3. **Identify gaps** — services where MFA isn't enabled or requires a tier upgrade
4. **SSO consolidation** — which apps support "Sign in with Google/Microsoft"?
5. **Shadow IT sweep** — check browser history, expense reports for unapproved tools
6. **Documentation** — prepare evidence for the Cyber Essentials assessor

Publish on LinkedIn with the headline: *"April 2026 Cyber Essentials Update: MFA on EVERYTHING or You Fail. Here's the Checklist."*

Gate the PDF behind an email capture form → build a lead list of UK businesses scrambling to comply.
