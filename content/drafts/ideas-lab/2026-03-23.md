## 1. Infrastructure & Automation

### Top Idea: Deploy Flash-MoE for Blazing Local Inference on Mac Mini

**Flash-MoE** is the most technically impressive local LLM project to land this year. It runs Qwen3.5-397B-A17B (a 397 billion parameter MoE model) on a MacBook Pro with 48GB RAM at 4.4+ tokens/second. The entire 209GB model streams from SSD through custom Metal compute shaders. No Python, no frameworks — just C, Objective-C, and hand-tuned Metal kernels.

**Why this matters for Kryptonome:** Your Mac mini M2 Pro has 32GB unified memory. While 397B needs 48GB, the technique is directly applicable to smaller MoE models. The key innovations — SSD expert streaming, FMA-optimized dequant kernels, "trust the OS" page cache strategy — could dramatically improve inference speed for any local model. The 2-bit mode runs even faster (5.74 tok/s) at 120GB on disk, though it breaks JSON/tool calling.

**The deeper insight:** Flash-MoE proves that Apple Silicon's unified memory + fast NVMe SSD is an underexplored architecture for LLM inference. The "stream experts from SSD" pattern means model size is bounded by disk, not RAM. Your Mac mini with a 1TB+ SSD could run models previously considered impossible.

**Revenue Potential:** Not direct, but this technique could make your Mac mini viable for local inference services alongside the GPU server. Offer "Apple Silicon inference consulting" — many enterprises are sitting on fleets of Mac Studios and don't know how to use them for AI.

### Quick Win: Clone and Benchmark Flash-MoE on Mac Mini M2 Pro

Even if you can't run the full 397B (48GB needed), clone the repo and study the Metal shader implementations. The FMA dequant kernel technique, SSD streaming pattern, and page cache strategy are all portable to other inference engines. `git clone https://github.com/danveloper/flash-moe.git && cd metal_infer && make` — 15 minutes to build, then experiment with smaller MoE models using the same techniques.
