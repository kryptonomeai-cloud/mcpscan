# Improvement Proposal — 2026-03-19

## Problem
**Daily MiniClaw Backup cron (`775b0bef`) has 3 consecutive timeout failures.** Last error: "All models failed (2): anthropic/claude-sonnet-4-6: LLM request timed out. | anthropic/claude-opus-4-6: LLM request timed out." Duration: 1,214,850ms (~20 min). The backup script likely hangs on iCloud operations (same root cause as the iCloud cleanup failures from Mar 16-19). This means workspace backups haven't succeeded in 3+ days.

## Proposed Fix
1. Add `timeout 120` wrappers around all iCloud copy/cleanup operations in `scripts/backup.sh`
2. Increase the cron `timeoutSeconds` from default to 300 (5 min should be plenty for a backup)
3. Add a fallback: if iCloud copy fails, copy to a local `/tmp/` staging path and log the failure instead of hanging
4. Set delivery mode to `announce` so failures are visible to the user

## Impact
- Restores daily workspace backups (currently broken 3 days)
- Prevents 20-min timeout burns (~$0.50+/run in wasted API calls)
- Fixes the same iCloud hang pattern affecting paperless cleanup

## Risk
- **LOW** — only adds timeouts and fallback paths to existing backup script. No data deletion. Worst case: backup still fails but now fails fast instead of hanging.

## Implementation
1. Read `scripts/backup.sh` and identify iCloud FS operations
2. Wrap each in `timeout 120`
3. Add fallback error handling (copy to /tmp if iCloud hangs)
4. Update cron job `775b0bef` with `timeoutSeconds: 300`
5. Test manually: `bash scripts/backup.sh`
