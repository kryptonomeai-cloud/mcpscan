# Improvement Proposal — 2026-03-21

## Problem
**This self-improvement cron has generated 11 proposals over 11 days with ZERO implementations.** It runs daily on Opus (most expensive model), reads ~300 lines of memory, scans session logs, writes proposals, and updates trackers — all for zero measurable output. At ~$0.30-0.50/run, it's burned ~$4-5 on bureaucracy. The iCloud backup fix (proposed Mar 19) remains unapplied. The pipeline doesn't lack ideas — it lacks an approval mechanism.

## Proposed Fix
**Reduce this cron to weekly (Saturday only) and switch to Sonnet model.** Daily runs are generating diminishing returns (same problems rediscovered daily). Weekly cadence:
1. Change cron schedule from `0 5 30 * * *` to `0 5 30 * * 6` (Saturdays only)
2. Switch model from Opus to Sonnet (sufficient for analysis + writing)
3. Batch the week's proposals into a single user-facing summary requesting approval
4. Add a direct Telegram prompt: "Reply ✅ to approve, ❌ to reject"

## Impact
- Saves ~$3/week in token costs (6 fewer Opus runs)
- Reduces noise — user gets one weekly summary instead of daily proposals they never see
- Forces batched review — higher chance of actually getting approvals

## Risk
- **LOW** — Reducing frequency of a zero-throughput process has no downside. Can revert to daily if weekly proves insufficient.

## Implementation
1. Update cron schedule: `cron update --id 30d6d23c --schedule '0 5 30 * * 6'`
2. Update cron model to sonnet
3. Modify report format to include approval prompt
4. Consolidate all 11 pending proposals into a single ranked list for user review
